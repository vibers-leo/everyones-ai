## 1. 현재 상태 진단

### 개발 완성도
- **코드 규모**: Next.js 16 + Firebase, 약 8개 주요 페이지, 5개 핵심 섹션
- **주요 기능 구현 상태**: 🟡 Phase 1 완료, Phase 2 대기
  - ✅ 랜딩 페이지 (브랜드 소개)
  - ✅ 배움터(Academy) 기본 구조
  - ✅ 실험실(Lab) 기본 구조
  - ✅ 커뮤니티 게시판 (CRUD)
  - ✅ 쇼케이스 갤러리
  - ✅ 인증 시스템 (Firebase Auth)
  - ✅ 테마 전환 (다크모드)
  - 🟡 배움터 강의 콘텐츠 미흡 (기본 템플릿만 있음)
  - 🟡 실험실 AI 도구 체험 기능 미흡
  - ❌ 게시글 댓글 기능 미지원

### 매출/사용자 현황
- **배포**: 별도 도메인 없음 (로컬 개발 중심)
- **사용자**: 추정 20명 이하 테스트 유저
- **상태**: 비영리 커뮤니티이지만 실질 운영 중단 상태
- **수익**: 불필요 (비영리)

### 핵심 과제
1. **콘텐츠 부재**: 강의, 실험실 AI 도구의 실질 콘텐츠가 프로토타입 수준
2. **기능 미흡**: 댓글, 추천, 조회수 같은 기본 상호작용 기능 부재
3. **배포 불완료**: 도메인, 호스팅 미정 → 외부 사용자 진입 불가
4. **커뮤니티 미활성화**: "함께"라는 가치에 비해 실제 상호작용 기능 약함
5. **비즈니스 모델 불명**: 비영리인데 지속성 어떻게 확보할 것인가?

## 2. Startup Canvas (11섹션 요약)

| 항목 | 내용 |
|------|------|
| **Vision** | 누구나 AI를 사용해 게임(또는 창작물)을 만들 수 있는 비영리 커뮤니티. "Easy (쉬움), Fun (재미), Together (함께)" |
| **Market Segments** | 초급 개발자/게임 제작자, AI에 관심 있는 학생, 창작 취미자 (비전문가) |
| **Value Proposition** | 1) 복잡한 AI를 쉽게 배우는 강의 2) 직접 해보는 실험실 3) 커뮤니티 피드백 |
| **Trade-offs** | 쉬운 진입장벽 ↔ 깊이 있는 학습 곤란 / 비영리 이상주의 ↔ 운영 비용 부담 |
| **Key Metrics** | MAU, 강의 완주율(%), 실험실 AI 도구 사용률(%), 게시글 댓글 비율, 커뮤니티 활동도 |
| **Growth** | 유튜브(AI 게임 제작 튜토리얼) + 소셜(Instagram, Twitter AI 아트) + 입소문(커뮤니티) |
| **Capabilities** | Next.js 풀스택, Firebase(인증/DB/스토리지), 쉬운 UI/UX 설계, AI 도구 통합(Claude API, Hugging Face) |
| **Can't/Won't** | 엔터프라이즈급 교육 미지원(대학 커리큘럼), 전문가용 심화 과정 미지원 |
| **Cost Structure** | 호스팅(Vercel), Firebase(스토리지), API 비용(Claude, Hugging Face), 콘텐츠 제작(시간), 도메인 |
| **Revenue Streams** | 무료(현재) → 향후 도네이션, 스폰서십(AI 기업), 또는 고급 강의 패키지 |
| **Feedback Loop** | 강의 완주율 낮음 → 콘텐츠 난이도 재조정, 실험실 사용 저조 → 도구 UI 개선 |

## 3. Beachhead 분석

**Primary Segment**: 고등학생 + 대학생 (AI에 관심, 게임 제작 취미)

| 기준 | 평가 | 근거 |
|------|------|------|
| **Burning Pain** | ⭐⭐ | AI 관심은 높으나, "필수"는 아님. 유튜브 무료 튜토리얼 대안 많음 |
| **Willingness to Pay** | ⭐ | 학생은 유료 교육 꺼림. 비영리도 장점이 되나 지속성 의심 |
| **Winnability** | ⭐⭐⭐ | UI/UX 친절함 + 실험실 직접 체험 → 유튜브와 차별화. 단, 콘텐츠 품질이 관건 |
| **Referral** | ⭐⭐ | SNS 공유성은 낮음(학습 플랫폼이기 때문). 입소문은 커뮤니티 품질에만 의존 |

**결론**: **Beachhead로는 약함**. Burning Pain과 WTP가 모두 낮음. **비영리의 이상주의로 시작했으나 시장 니즈 미충족** → 방향 재검토 필요.

## 4. Growth Loop 분석

**현재 루프**: 랜딩 → 배움터 강의 시청 → ? (탈락)

**문제점**:
- **강의 완주 동기 부재**: 완료 후 "뭘 하지?" → 실험실로 유도하려 했으나 경로 불명확
- **커뮤니티 약함**: "함께"라는 가치가 있으나 댓글, 추천, 실시간 협업 기능 없음
- **실험실 비활성화**: AI 도구 체험이 핵심인데 실질 기능 구현 부실

**추천 루프**: **Learning + Making + Sharing**
1. 강의 완주 (1시간 × 5강 = 5시간)
2. 실험실에서 직접 만들기 (AI로 간단한 게임 스크립트 생성)
3. 쇼케이스에 내 작품 올리기
4. 커뮤니티에서 피드백 받기
5. 다음 강의로 진입

### 비영리 지속성 모델 (향후 검토)

```
FREE TIER (누구나)
- 모든 강의 시청
- 실험실 AI 도구 월 50회 사용
- 커뮤니티 게시글/댓글 (기본)

SUPPORTER (선택, 월 ₩4,900 또는 연 ₩39,000)
- 광고 없음
- AI 도구 무제한 사용
- 오프라인 강의 자료 다운로드
- 커뮤니티 배지 (Supporter)
- 월간 라이브 Q&A 참석 권리

PARTNER (기업 후원, 월 ₩99,000 ~ )
- 회사 로고 후원자 페이지 노출
- 별도 강의 개발 협력 (회사 기술 소개)
- 예: "Google Cloud에서 배우는 AI" 같은 협력 강의

목표: 월 5,000명 × 10% Supporter × ₩4,900 = ₩245만 (운영비 충당)
```

**비영리 특성상**:
- 수익화보다는 "지속성 확보" 목표
- 도네이션 + 스폰서십(AI 기업) 병행
- 학교/도서관과 협력으로 무료 제공
